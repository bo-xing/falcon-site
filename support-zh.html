使用 Falcon 遇到问题？从这里开始。

Falcon 是一款基于 iperf 的 iOS 网络吞吐测试工具。如果你遇到问题或有疑问，欢迎发邮件给我们——每一封都会认真查看。

为便于我们复现问题，来信请尽量附上：iOS 版本、设备型号、使用的引擎（iperf3 / iperf2），以及这次测试是本地直连还是走 SSH。

常见问题

为什么测试连不上？

吞吐测试需要对端有一个正在运行的 iperf server。请确认目标设备在你的网络中可达，并且 iperf server 正监听你指定的端口（iperf3 默认 5201，iperf2 默认 5001）。使用 Falcon Pro 时，SSH 编排功能会自动帮你起停对端的 server。

免费版和 Pro 版有什么区别？

免费版可以完整手动跑吞吐测试（TCP / UDP、上传 / 下载方向、并行流、自定义码率），并查看、导出、分享你的全部历史记录。Falcon Pro（一次性买断）额外提供：SSH 远程编排、命令模式、自定义预设与内置场景、保存 / 收藏常用目标，以及历史运行对比和批量导出。

SSH 编排是做什么的？

不用再手动到对端敲 iperf -s。Falcon 会通过 SSH 连到对端设备（OpenWrt / Linux / 另一台机器），自动起好 iperf server，在手机上跑 client，测完再自动回收对端进程。支持密码与私钥认证，并对主机密钥做校验（TOFU）。

我的 UDP 测试没有抖动 / 丢包数据？

抖动和丢包来自对端 server 的回传报告。在某些 UDP 测试方向下（尤其是 iperf2 正向 UDP），这些数值可能拿不到——此时 Falcon 会显示「—」并提示「未收到对端确认」。界面上显示的速度是发送端口径的码率。

Falcon 会收集我的数据吗？

不会。所有测试结果都只保存在你的设备上，SSH 凭据仅存于 iOS 钥匙串（Keychain），不会回传给开发者。详见隐私政策。

换了新设备如何恢复购买？

打开 Falcon 的「设置」页，找到 Falcon Pro 区域，点击「恢复购买」。只要使用同一个 Apple ID 登录，你的 Pro 解锁状态就会恢复。
